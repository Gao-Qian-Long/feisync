# 用户文档与配置指南

## 概述

Obsidian-Flybook 插件将本地 Obsidian 笔记作为**普通文件**同步到飞书 Drive，实现备份和多端访问。

## 功能特点

- 支持指定文件夹同步
- 手动或自动同步（基于文件变化）
- 以本地文件为准，单向同步
- 上传为普通文件（非云文档）

## 安装

### 通过社区插件市场

1. 打开 Obsidian 设置 → 社区插件 → 浏览
2. 搜索 "Flybook"
3. 点击安装，然后启用

### 手动安装

1. 下载插件文件
2. 复制到 `<vault>/.obsidian/plugins/obsidian-flybook/`
3. 重启 Obsidian 并启用插件

## 配置飞书应用

### 1. 创建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 创建企业自建应用
3. 进入「凭证与基础信息」获取 App ID 和 App Secret

### 2. 添加权限

在「权限管理」页面添加以下权限：

| 权限名称 | 权限标识 |
|----------|----------|
| 上传、下载文件或图片 | `drive:drive` 或 `drive:file` |

### 3. 发布应用

创建版本并发布应用，权限才会生效。

## 插件设置

### 基础配置

| 字段 | 说明 |
|------|------|
| 飞书 App ID | 飞书应用的 App ID |
| 飞书 App Secret | 飞书应用的 App Secret |
| 代理服务器地址 | 用于中转 API 请求（可选） |

### 文件夹配置

| 字段 | 说明 |
|------|------|
| 本地同步文件夹 | 相对于仓库根目录的路径，如 `Notes` |
| 飞书目标文件夹 Token | 可选，留空则使用默认文件夹 |

### 同步配置

| 字段 | 说明 |
|------|------|
| 自动同步 | 监听文件变化自动上传 |
| 同步间隔（分钟） | 防抖间隔 |

## 使用方法

### 手动同步

1. 点击左侧 Ribbon 图标 → 立即同步
2. 或使用命令面板（Ctrl+P）→ `Flybook: Sync now`

### 自动同步

1. 在设置中开启「自动同步」
2. 设置同步间隔
3. 修改文件后，自动在指定间隔后同步

## 同步流程

1. 扫描本地文件夹
2. 在飞书中创建/定位目标文件夹
3. 逐个上传文件
4. 如果云端存在同名文件，先删除再上传

## 支持的文件类型

| 类型 | 扩展名 |
|------|--------|
| 文本文件 | `.md`, `.txt`, `.json`, `.yml` 等 |
| 二进制文件 | `.pdf`, `.docx`, `.xlsx`, `.png`, `.jpg` 等 |

## 文件大小限制

单个文件 ≤20MB

## 故障排除

### 测试连接失败

- 检查 App ID 和 App Secret 是否正确
- 确认网络可以访问飞书开放平台
- 如使用代理，确认代理服务器正常运行

### 文件上传失败

- 检查文件大小是否超过 20MB
- 确认飞书云空间充足

### 自动同步未触发

- 确认「自动同步」已开启
- 检查文件是否在配置的同步文件夹内

## 代理服务器

如果无法直接访问飞书 API，需要搭建代理服务器。详见 `plans/proxy-server-setup.md`

## 附录

### 获取文件夹 Token

1. 在飞书 Drive 进入目标文件夹
2. 地址栏 URL 类似 `https://xxx.feishu.cn/drive/folder/xxx`
3. `xxx` 部分即为文件夹 Token
